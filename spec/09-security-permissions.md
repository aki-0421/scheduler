# セキュリティ・権限仕様

## 1. 基本方針

このアプリはローカル Mac 上で Codex CLI にプロンプトを渡し、必要に応じて shell command やファイル編集を伴う処理を実行させる。そのため、安全側の default、明示的な project trust、audit log、権限 token、worktree 隔離を必須とする。

## 2. Trust boundary

```text
Trusted:
- User operating macOS account
- Codex Scheduler app bundle
- codex-schedulerd sidecar
- codex-schedule bundled CLI
- SQLite DB under app support directory

Conditionally trusted:
- User-selected repository paths
- Codex CLI binary path
- Git binary

Untrusted / must validate:
- Prompt text generated by model
- Arguments passed by scheduled Codex session to codex-schedule
- Repository contents
- Shell output
- Web/search content consumed by Codex
```

## 3. Project trust

Repository/folder task は trust された path のみ保存できる。

Trust flow:

1. UI folder picker で path 選択。
2. canonical path を計算。
3. Git root を検出。
4. warning を表示。
5. user が trust する。
6. `projects.trusted_at` に保存。

Untrusted path を CLI が指定した場合:

- human terminal なら confirmation required error。
- scheduled Codex session なら task を `paused` + `review_required` として保存、または reject。

## 4. Codex CLI path trust

Codex binary path は onboarding で確認する。

Checks:

- path exists。
- executable bit。
- `codex --version` success。
- symlink chain を表示。
- path が user writable location の場合 warning。

## 5. Sandbox defaults

| target | default sandbox | approval |
| --- | --- | --- |
| chat | read-only | never |
| repo-local | workspace-write | never |
| repo-worktree | workspace-write | never |

`danger-full-access`:

- advanced only。
- confirmation dialog required。
- scheduled session CLI からは default で作成不可。
- task list に常時 warning badge。

## 6. Approval policy

MVP の scheduled run は非対話実行を基本とするため、default は `never`。

理由:

- 予定実行はユーザーが見ていない時間に走る。
- 承認待ちで run が永久停止する UX を避ける。
- 必要な操作は sandbox / allowlist / project trust で事前に制御する。

将来:

- UI で permission prompt を受ける interactive scheduled run。
- paused-until-approved run state。

## 7. Schedule CLI capability token

Scheduled run に渡す token は次の性質を持つ。

- run ごとにランダム生成。
- DB には hash のみ保存。
- expires_at は run 終了 + 1 時間、または発行から最大 24 時間。
- capability list を持つ。
- max create count を持つ。
- revoked_at により無効化可能。

Token は environment variable `CODEX_SCHEDULER_RUN_TOKEN` で渡すが、ログには保存しない。

## 8. Capability 最小権限

Default:

```json
[
  "schedule:list",
  "schedule:create",
  "schedule:update-current",
  "schedule:pause-current"
]
```

危険 capability:

```text
schedule:update-any
schedule:delete
schedule:run-now
schedule:create-danger-full-access
schedule:trust-project
```

危険 capability は UI で明示 opt-in。

## 9. CLI abuse mitigation

Scheduled Codex session が過剰に schedule を作ることを防ぐ。

- per-run create limit: default 5。
- per-task daily create limit: default 20。
- global active task limit: default 1,000。
- `* * * * *` のタスク作成は allowed だが warning audit を残す。
- nested self-replicating prompt を検出する heuristic を将来追加。

## 10. Shell injection 対策

- Codex command は shell 経由ではなく argv 配列で起動。
- Git command も argv 配列で起動。
- prompt や task name を shell command に埋め込まない。
- log display は HTML escape する。
- file path は canonicalize する。

## 11. Worktree safety

- worktree root は app data 配下。
- branch name は sanitized slug + run id。
- existing path には書き込まない。
- cleanup 前に `git status --porcelain` を確認。
- uncommitted changes がある worktree は自動削除しない。
- repo-local mode の場合は実行前に dirty status を snapshot し、UI に表示する。

## 12. Data privacy

保存するもの:

- task prompt。
- run final output。
- stdout/stderr。
- Codex JSONL events。
- Git diff/stat。

保存しない、または redact するもの:

- API key。
- run token。
- secret-like env vars。
- full environment without redaction。

ユーザー操作:

- task delete。
- run log delete。
- all data reset。
- diagnostic export redaction preview。

## 13. Network

本アプリ自体は外部ネットワーク通信をしない。ただし Codex CLI は自身の仕様に従って OpenAI と通信する。アプリは Codex CLI の通信内容を proxy しない。

## 14. Notifications and privacy

macOS notification には sensitive prompt/full output を出さない。

Good:

```text
Daily PR Review succeeded. Open Codex Scheduler to review results.
```

Avoid:

```text
Found secret: sk-...
```

## 15. Audit log

以下を記録する。

- task create/update/delete/pause/resume。
- schedule CLI による変更。
- scheduled run による task create/update。
- dangerous sandbox 有効化。
- project trust/untrust。
- run cancel。
- cleanup worktree。

Audit event はユーザーが UI で確認できる。

## 16. Prompt injection consideration

Codex が読む repository や web content には「新しい schedule を作れ」などの悪意ある指示が含まれうる。対策:

- system instructions で「ユーザーの依頼と無関係な schedule は作らない」と明記。
- capability token で上限を設ける。
- 不確実な task は `--paused` を推奨。
- created_by_run_id を UI に表示し、レビュー可能にする。
- 将来、new schedule created by model は default review queue に入れる設定を追加。

## 17. macOS permissions

必要になりうる権限:

- Notifications。
- Files and Folders / Full Disk Access は原則要求しない。ユーザー選択 folder のみ扱う。
- Login Items は optional。
- Prevent sleep は optional。実装する場合、明示 setting と indicator を用意する。

## 18. Secure defaults checklist

- [ ] New repo task は worktree default。
- [ ] Approval policy default は `never`。
- [ ] Sandbox default は `read-only` or `workspace-write`。
- [ ] danger-full-access は advanced only。
- [ ] run token は redacted。
- [ ] CLI create limit を enforce。
- [ ] untrusted repo path は reject or paused。
- [ ] all writes audit logged。
- [ ] logs are escaped in UI。
- [ ] worktree cleanup does not delete dirty worktree。
